## Stats filter question

**Does `/stats/field/values?fields=Phase` accept an additional `query.cond=melanoma` and change the returned counts? -> NO.**

The endpoint returns **HTTP 400** when `query.cond=melanoma` is added — it rejects the search expression outright (the `query.*` params are not valid on `/stats/field/values`). **Implication:** filtered distributions cannot be obtained from the stats endpoint at all; they must be computed client-side by paging `/studies`. This confirms the assumption in system-design §6.

## Paging / pageSize behavior

- `pageSize` caps results per page (max 1000; default 10 — always set it). This spike used `pageSize=50` and received 50 study records on the first page.
- `nextPageToken` present on the first page: **True**. Pass it back as `pageToken` to fetch the next page; absence means the result set fit in one page.
- `countTotal=true` returns `totalCount` for the whole filtered result set independent of `pageSize`.
